## Updating the PWA app icons (home screen icons)

The icons you see when someone saves a tool to their iOS/Android home screen are generated by a script — not edited manually.

### Icon script location
`C:\Users\andyg\Desktop\cursor\mmg-tools\scripts\generate-pwa-icons.ps1`

### What the script does
Generates 3 icon sizes (512px, 192px, 180px) for each of the 3 tools. Output goes to:
- `mmg-tools\guide\icons\`
- `mmg-tools\calculator\icons\`
- `mmg-tools\day-cost\icons\`

### To change icon appearance (colours, text, layout)
Edit the `$apps` array at the top of the script — change `Background`, `Foreground`, `Accent`, or `Label` values.

### To regenerate icons after changes
Tell Claude: "regenerate PWA icons" — Claude will run the script and push the new icons to GitHub.

Or run manually:
```powershell
& "C:\Users\andyg\Desktop\cursor\mmg-tools\scripts\generate-pwa-icons.ps1"
```
Then push to GitHub as normal.

- `MMG_Logo_White_Transparent.png` — used by day-cost (dark green background)
- `MMG_Logo_Grey_Transparent.png` — used by calculator (black background)

## Publishing a blog post on mrmallorcagolf.com

Blog posts are guide pages that live in `src/app/guides/[slug]/`.

### Steps
1. Tell Claude: "Write a blog post about [topic]" — use the `blog-writing` and `seo-content` skills
2. Claude creates the page file in `src/app/guides/[slug]/page.jsx`
3. Claude adds translations for all 7 languages
4. Run checks: `npm run check:text && npm run check:i18n-release && npm run build`
5. Push to GitHub → Vercel auto-deploys to mrmallorcagolf.com in ~60 seconds

### Languages
English is always written first. Translations generated for: de, es, fr, nl, sv, zh.
Never add content to a language page that isn't in the English version.

## Adding a new HTML tool (future)

1. Create folder in `C:\Users\andyg\Desktop\cursor\mmg-tools\` (e.g. `new-tool/`)
2. Copy in the HTML as `index.html`
3. Copy `netlify.toml` from any existing folder
4. Add OG meta tags to the `<head>`
5. Add icon config to `scripts\generate-pwa-icons.ps1` and run it
6. Push to GitHub
7. New Netlify site → Import from GitHub → base dir: `new-tool`
8. Add custom domain in Netlify + CNAME in Cloudflare
9. Save a copy to Downloads/Apps for easy editing
